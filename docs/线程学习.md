# 线程学习：第一部分

这份笔记整理 C++ 标准线程库的第一部分内容，重点包括：

```text
启动线程
join 和 detach
异常情况下等待线程
RAII 管理线程
后台运行线程
线程参数传递
std::ref
std::move 和线程所有权转移
std::thread::hardware_concurrency()
std::thread::id
```

使用线程需要包含头文件：

```cpp
#include <thread>
```

## 1. 什么是线程

线程是程序中的一条执行路径。

每个 C++ 程序至少有一个线程，也就是执行 `main()` 函数的主线程。使用 `std::thread` 可以创建新的线程，让其他函数和主线程同时运行。

示例：

```cpp
#include <iostream>
#include <thread>

void work()
{
    std::cout << "worker thread\n";
}

int main()
{
    std::thread t(work);
    t.join();

    std::cout << "main thread\n";
}
```

这句代码会启动新线程：

```cpp
std::thread t(work);
```

可以理解为：

```text
构造 std::thread 对象 = 启动一个新线程
```

## 2. 启动线程

### 2.1 用普通函数启动线程

```cpp
void do_some_work()
{
    // 线程任务
}

std::thread my_thread(do_some_work);
```

`my_thread` 是一个 `std::thread` 对象，用来管理新启动的线程。

### 2.2 用函数对象启动线程

只要一个类重载了 `operator()`，它的对象就可以像函数一样调用，这种对象叫函数对象。

```cpp
#include <iostream>
#include <thread>

class BackgroundTask
{
public:
    void operator()() const
    {
        std::cout << "do something\n";
        std::cout << "do something else\n";
    }
};

int main()
{
    BackgroundTask task;
    std::thread t(task);
    t.join();
}
```

`std::thread` 会把函数对象复制到新线程的内部存储中，然后在线程中调用它。

### 2.3 最令人头痛的语法解析

下面这句容易出问题：

```cpp
std::thread my_thread(BackgroundTask());
```

你可能以为它是在创建一个临时 `BackgroundTask` 对象并启动线程，但 C++ 可能把它解析成一个函数声明。

解决办法：

```cpp
std::thread my_thread((BackgroundTask()));
std::thread my_thread{BackgroundTask()};
```

更推荐使用统一初始化：

```cpp
std::thread my_thread{BackgroundTask()};
```

### 2.4 用 lambda 启动线程

```cpp
std::thread t([] {
    std::cout << "hello from lambda thread\n";
});

t.join();
```

lambda 很适合写短小线程任务，也能捕获局部变量。但捕获引用时必须特别注意生命周期。

## 3. join 和 detach

线程启动后，在 `std::thread` 对象销毁前必须做出选择：

```text
join：等待线程结束。
detach：让线程独立在后台运行。
```

如果一个 `std::thread` 对象析构时仍然管理着一个线程，并且没有调用 `join()` 或 `detach()`，程序会调用：

```cpp
std::terminate()
```

### 3.1 join

```cpp
std::thread t(work);
t.join();
```

`join()` 的含义是：

```text
当前线程等待 t 管理的线程执行完。
```

适合需要确保工作线程完成之后再继续执行的场景。

### 3.2 detach

```cpp
std::thread t(work);
t.detach();
```

`detach()` 的含义是：

```text
让线程脱离 std::thread 对象，独立在后台运行。
```

调用 `detach()` 后：

```cpp
t.joinable() == false
```

此时不能再对这个线程调用 `join()`。

## 4. detach 的生命周期风险

分离线程最大的风险是：后台线程还在运行，但它访问的数据已经销毁。

危险示例：

```cpp
#include <thread>

void do_something(int & value)
{
    ++value;
}

struct Func
{
    int & i;

    Func(int & value) : i(value) {}

    void operator()()
    {
        for (unsigned j = 0; j < 1000000; ++j)
        {
            do_something(i);
        }
    }
};

void oops()
{
    int some_local_state = 0;
    Func my_func(some_local_state);

    std::thread t(my_func);
    t.detach();
}
```

问题：

```text
some_local_state 是 oops() 的局部变量。
Func 内部保存的是 some_local_state 的引用。
detach 后，oops() 不等待线程结束。
oops() 返回后，some_local_state 被销毁。
后台线程可能还在通过引用 i 访问它。
```

所以危险不是 `i` 被销毁，而是：

```text
i 引用的对象 some_local_state 被销毁。
```

此时 `i` 变成悬空引用。

安全做法之一是使用 `join()`：

```cpp
void safe()
{
    int some_local_state = 0;
    Func my_func(some_local_state);

    std::thread t(my_func);
    t.join();
}
```

这样 `safe()` 返回前，线程一定已经结束，局部变量仍然有效。

## 5. 异常情况下等待线程

普通写法有风险：

```cpp
void f()
{
    int some_local_state = 0;
    Func my_func(some_local_state);

    std::thread t(my_func);

    do_something_in_current_thread(); // 如果这里抛异常

    t.join(); // 可能执行不到
}
```

如果 `do_something_in_current_thread()` 抛异常，`t.join()` 会被跳过。随后 `t` 析构时仍然 joinable，程序会调用 `std::terminate()`。

手动处理方式：

```cpp
void f()
{
    int some_local_state = 0;
    Func my_func(some_local_state);

    std::thread t(my_func);

    try
    {
        do_something_in_current_thread();
    }
    catch (...)
    {
        t.join();
        throw;
    }

    t.join();
}
```

这样：

```text
正常路径：最后 join。
异常路径：catch 中先 join，再重新抛出异常。
```

## 6. 用 RAII 管理线程：thread_guard

手动写 `try/catch` 容易遗漏，更好的方式是 RAII。

RAII 思想：

```text
对象构造时接管资源。
对象析构时释放资源。
```

线程守卫类：

```cpp
#include <thread>

class thread_guard
{
private:
    std::thread & t;

public:
    explicit thread_guard(std::thread & t_)
        : t(t_)
    {
    }

    ~thread_guard()
    {
        if (t.joinable())
        {
            t.join();
        }
    }

    thread_guard(const thread_guard &) = delete;
    thread_guard & operator=(const thread_guard &) = delete;
};
```

使用：

```cpp
void f()
{
    int some_local_state = 0;
    Func my_func(some_local_state);

    std::thread t(my_func);
    thread_guard guard(t);

    do_something_in_current_thread();
}
```

当 `f()` 结束时，不管是正常结束还是异常退出，`guard` 都会析构，并在析构函数中调用 `join()`。

为什么要删除复制：

```cpp
thread_guard(const thread_guard &) = delete;
thread_guard & operator=(const thread_guard &) = delete;
```

因为多个 `thread_guard` 管理同一个线程很危险。第一个 guard `join()` 后，第二个 guard 再 `join()` 就不合法。

## 7. 后台运行线程

调用 `detach()` 后，线程在后台独立运行：

```cpp
std::thread t(do_background_work);
t.detach();
```

分离后：

```cpp
assert(!t.joinable());
```

适合后台任务：

```text
后台日志写入
后台缓存清理
后台文件监控
独立窗口处理
不需要等待结果的任务
```

但必须保证：

```text
后台线程访问的数据，在它运行期间一直有效。
```

例如，尽量避免这样：

```cpp
void f()
{
    int local = 10;

    std::thread t([&] {
        do_something(local);
    });

    t.detach();
}
```

因为 `local` 可能在线程使用前已经销毁。

更安全的是按值捕获：

```cpp
void f()
{
    int local = 10;

    std::thread t([local] {
        do_something(local);
    });

    t.detach();
}
```

## 8. 传递线程参数

`std::thread` 构造函数可以接受函数参数：

```cpp
void f(int i, const std::string & s);

std::thread t(f, 3, "hello");
```

这表示新线程会调用：

```cpp
f(3, "hello");
```

重要规则：

```text
std::thread 默认会把传入参数复制或移动到新线程自己的内部存储中。
```

## 9. 字符数组参数的危险

危险示例：

```cpp
#include <cstdio>
#include <string>
#include <thread>

void f(int i, const std::string & s);

void oops(int some_param)
{
    char buffer[1024];
    std::sprintf(buffer, "%i", some_param);

    std::thread t(f, 3, buffer);
    t.detach();
}
```

虽然 `std::thread` 会复制参数，但这里复制的是指针，不是数组内容。

原因：

```text
buffer 是 char[1024]。
作为函数参数传递时，数组会退化成 char*。
std::thread 内部保存的是 char* 指针值。
```

如果 `oops()` 返回后，局部数组 `buffer` 被销毁，而新线程才开始用这个 `char*` 构造 `std::string`，就会访问已经销毁的内存。

安全写法：

```cpp
void not_oops(int some_param)
{
    char buffer[1024];
    std::sprintf(buffer, "%i", some_param);

    std::thread t(f, 3, std::string(buffer));
    t.detach();
}
```

这里先在当前线程中构造 `std::string`，然后 `std::thread` 复制这个字符串对象到新线程内部。

## 10. 传引用参数：std::ref

如果线程函数需要非常量引用：

```cpp
void update_data_for_widget(widget_id w, widget_data & data);
```

不能直接写：

```cpp
widget_data data;
std::thread t(update_data_for_widget, w, data);
```

因为 `std::thread` 默认会复制参数。

正确写法：

```cpp
#include <functional>

widget_data data;
std::thread t(update_data_for_widget, w, std::ref(data));
```

`std::ref(data)` 表示：

```text
不要复制 data，而是在线程函数中传递 data 的引用。
```

使用引用传参时，必须保证被引用对象在线程结束前仍然存在：

```cpp
std::thread t(update_data_for_widget, w, std::ref(data));
t.join();
process_widget_data(data);
```

## 11. 传递成员函数

可以把成员函数作为线程函数：

```cpp
class X
{
public:
    void do_lengthy_work()
    {
        // ...
    }
};

X my_x;
std::thread t(&X::do_lengthy_work, &my_x);
```

这表示新线程调用：

```cpp
my_x.do_lengthy_work();
```

如果成员函数有参数：

```cpp
class X
{
public:
    void do_lengthy_work(int value)
    {
        // ...
    }
};

X my_x;
int num = 42;

std::thread t(&X::do_lengthy_work, &my_x, num);
```

等价于在新线程中调用：

```cpp
my_x.do_lengthy_work(num);
```

## 12. 传递只能移动的对象

`std::unique_ptr` 不能复制，只能移动。

示例：

```cpp
#include <memory>
#include <thread>

class big_object
{
public:
    void prepare_data(int value)
    {
        // ...
    }
};

void process_big_object(std::unique_ptr<big_object> ptr)
{
    // ptr 拥有 big_object
}

int main()
{
    std::unique_ptr<big_object> p(new big_object);
    p->prepare_data(42);

    std::thread t(process_big_object, std::move(p));
    t.join();
}
```

所有权转移过程：

```text
p 拥有 big_object
std::move(p)
std::thread 内部存储拥有 big_object
新线程启动
process_big_object 的参数拥有 big_object
函数结束
unique_ptr 析构并 delete big_object
```

`std::move(p)` 之后：

```cpp
p == nullptr
```

## 13. std::thread 的所有权转移

`std::thread` 和 `std::unique_ptr` 类似：

```text
不可复制
可以移动
```

错误：

```cpp
std::thread t1(work);
// std::thread t2 = t1; // 错误，thread 不能复制
```

正确：

```cpp
std::thread t1(work);
std::thread t2 = std::move(t1);
```

移动后：

```text
t2 管理原来的线程。
t1 不再管理任何线程。
```

### 13.1 所有权转移示例

```cpp
void some_function();
void some_other_function();

std::thread t1(some_function);
std::thread t2 = std::move(t1);

t1 = std::thread(some_other_function);

std::thread t3;
t3 = std::move(t2);

// t1 = std::move(t3); // 危险：t1 已经管理一个线程，会调用 std::terminate()
```

给 `std::thread` 赋新线程之前，目标对象必须没有管理旧线程。

安全写法：

```cpp
if (t1.joinable())
{
    t1.join();
}

t1 = std::move(t3);
```

不能通过赋新值的方式丢弃一个仍在运行的线程。

## 14. 函数返回 std::thread

因为 `std::thread` 可以移动，所以可以从函数返回：

```cpp
std::thread make_thread()
{
    return std::thread(work);
}
```

也可以返回局部变量：

```cpp
std::thread make_thread_with_arg()
{
    std::thread t(work_with_arg, 42);
    return t;
}
```

调用：

```cpp
std::thread t = make_thread();
t.join();
```

## 15. std::thread 作为函数参数

可以按值接收 `std::thread`，因为它可以移动：

```cpp
void take_thread(std::thread t)
{
    t.join();
}

void f()
{
    take_thread(std::thread(work));

    std::thread t(work);
    take_thread(std::move(t));
}
```

不能写：

```cpp
take_thread(t);
```

因为这会尝试复制 `std::thread`。

## 16. scoped_thread：拥有线程并自动 join

`thread_guard` 保存的是外部 `std::thread` 的引用。`scoped_thread` 自己拥有一个 `std::thread` 对象。

```cpp
#include <stdexcept>
#include <thread>

class scoped_thread
{
private:
    std::thread t;

public:
    explicit scoped_thread(std::thread t_)
        : t(std::move(t_))
    {
        if (!t.joinable())
        {
            throw std::logic_error("No thread");
        }
    }

    ~scoped_thread()
    {
        t.join();
    }

    scoped_thread(const scoped_thread &) = delete;
    scoped_thread & operator=(const scoped_thread &) = delete;
};
```

使用：

```cpp
void f()
{
    int some_local_state = 0;
    scoped_thread t(std::thread(Func(some_local_state)));

    do_something_in_current_thread();
}
```

函数结束时，`scoped_thread` 自动析构，并调用 `join()`。

## 17. joining_thread 思路

`joining_thread` 是对 `std::thread` 的进一步封装：

```text
析构时自动 join。
赋值新线程前，如果已有线程，先 join。
支持移动，不支持复制。
```

核心思想：

```cpp
class joining_thread
{
private:
    std::thread t;

public:
    ~joining_thread()
    {
        if (t.joinable())
        {
            t.join();
        }
    }
};
```

C++20 提供了更现代的：

```cpp
std::jthread
```

它会在析构时自动请求停止并 join，比 `std::thread` 更安全。

## 18. 用 vector 管理多个线程

因为 `std::thread` 可移动，所以可以放进 `std::vector`。

```cpp
#include <thread>
#include <vector>

void do_work(unsigned id)
{
    // ...
}

void f()
{
    std::vector<std::thread> threads;

    for (unsigned i = 0; i < 20; ++i)
    {
        threads.emplace_back(do_work, i);
    }

    for (auto & entry : threads)
    {
        entry.join();
    }
}
```

注意：

```cpp
for (auto & entry : threads)
```

必须用引用。不能写：

```cpp
for (auto entry : threads)
```

因为 `std::thread` 不能复制。

## 19. 确定线程数量

可以使用：

```cpp
std::thread::hardware_concurrency()
```

获取硬件支持的并发线程数。

示例：

```cpp
unsigned count = std::thread::hardware_concurrency();
```

可能返回：

```text
4、8、16 等
```

如果无法获取，返回：

```cpp
0
```

所以要写备用值：

```cpp
unsigned hardware_threads = std::thread::hardware_concurrency();
unsigned thread_count = hardware_threads != 0 ? hardware_threads : 2;
```

线程数不是越多越好。线程太多会导致频繁上下文切换，反而降低性能。

## 20. 并行 accumulate 的思路

并行累加的核心思路：

```text
1. 计算总长度。
2. 根据最小任务量和硬件线程数决定线程数量。
3. 把区间拆成多个块。
4. 每个线程累加一块。
5. 主线程累加最后一块。
6. join 所有线程。
7. 汇总每块结果。
```

简化示例：

```cpp
#include <algorithm>
#include <numeric>
#include <thread>
#include <vector>

template <typename Iterator, typename T>
struct accumulate_block
{
    void operator()(Iterator first, Iterator last, T & result)
    {
        result = std::accumulate(first, last, result);
    }
};
```

函数对象调用：

```cpp
accumulate_block<Iterator, T> worker;
worker(block_start, last, results[index]);
```

这里：

```cpp
worker(...)
```

会调用重载的：

```cpp
operator()
```

等价于：

```cpp
worker.operator()(block_start, last, results[index]);
```

也可以创建临时对象并立即调用：

```cpp
accumulate_block<Iterator, T>()(
    block_start,
    last,
    results[index]
);
```

第一个括号创建临时对象，第二个括号调用 `operator()`。

## 21. std::advance 拆分区间

代码：

```cpp
Iterator block_end = block_start;
std::advance(block_end, block_size);
```

含义：

```text
把 block_end 从 block_start 开始向后移动 block_size 个元素。
```

例如总长度 100，`block_size = 25`：

```text
第 1 块：[0, 25)
第 2 块：[25, 50)
第 3 块：[50, 75)
最后块：[75, 100)
```

循环中每次：

```cpp
block_start = block_end;
```

让下一块从上一块结束的位置继续。

## 22. 线程 ID

线程 ID 类型是：

```cpp
std::thread::id
```

获取某个 `std::thread` 对象管理的线程 ID：

```cpp
std::thread t(work);
std::thread::id id = t.get_id();
```

获取当前正在执行代码的线程 ID：

```cpp
std::thread::id current = std::this_thread::get_id();
```

完整示例：

```cpp
#include <iostream>
#include <thread>

void work()
{
    std::cout << "worker id = "
              << std::this_thread::get_id() << '\n';
}

int main()
{
    std::cout << "main id = "
              << std::this_thread::get_id() << '\n';

    std::thread t(work);

    std::cout << "t.get_id() = "
              << t.get_id() << '\n';

    t.join();
}
```

区别：

```text
t.get_id()：获取 t 管理的线程 ID。
std::this_thread::get_id()：获取当前正在执行这句代码的线程 ID。
```

## 23. 用线程 ID 区分主线程和工作线程

```cpp
#include <iostream>
#include <thread>

std::thread::id main_thread_id;

void check_thread()
{
    if (std::this_thread::get_id() == main_thread_id)
    {
        std::cout << "main thread\n";
    }
    else
    {
        std::cout << "worker thread\n";
    }
}

int main()
{
    main_thread_id = std::this_thread::get_id();

    check_thread();

    std::thread t(check_thread);
    t.join();
}
```

`std::thread::id` 可以：

```text
比较
复制
输出
作为 std::map 的 key
作为 std::unordered_map 的 key
```

例如：

```cpp
#include <map>
#include <string>
#include <thread>

std::map<std::thread::id, std::string> thread_names;
```

## 24. 第一部分总结

```text
std::thread 对象创建时线程就启动。
线程对象销毁前必须 join 或 detach。
join 表示等待线程结束。
detach 表示线程后台独立运行。
detach 后要特别注意数据生命周期。
异常可能导致 join 被跳过，推荐使用 RAII。
```
